Visualization of Microscope Data

May 25, 2021 - 2 minutes read - 263 words

Dimensionality

Spatial

Current microscopes have the capability of capturing life and structures in the full dimensionality that they exist in. This is accomplished using digital sensors that reduce this complexity to a series of discrete values. How these values are interpreted turns into dimensionality. Take for example an image sensor. For each X and Y position there is an intensity value. These values are stored in sequence that if interrupted incorrectly will produce strange results, e.g. banding, stretching, wrapping, etc. Knowing the correct number of pixels in X and Y allows a computer to place each of these values appropriately on a grid.

To take this a step further, on a microscope, the focus plane can be moved axially (in and out in the direction of the lens / objective). Taking a series of 2-D images at different axial planes produces a 3-D volume. Knowing the size of the sensor in the above example allows the computer to place a new plane in Z creating 3-d pixels called voxels. Again, the computer stores a series of values in a linear fashion (like a long string). With size information, visualization tools can remap those values onto a grid corresponding to a physical location. Another dimension is time, where the microscope takes a series of planes or volumes separated by an amount of time. Dimensionality is abstractly an interpretation of how to split up the data.

Temporal

The third dimension could represent volume or time depending on how it is assigned. The last dimension is not as straight forward.
